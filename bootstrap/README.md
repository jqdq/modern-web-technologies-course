# CSS Framework Components

## Layout Components

1. **Grid System**
   - Description: Division of page into columns and rows for organized layout
   - Purpose: Creates responsive layouts that adapt to different screen sizes

2. **Container**
   - Description: Wrapper element that centers content and applies horizontal padding
   - Purpose: Maintains consistent margins and maximum width across viewport sizes

## Navigation Components

3. **Navbar**
   - Description: Horizontal navigation bar typically fixed at page top
   - Purpose: Provides primary site navigation and branding

4. **Sidebar**
   - Description: Vertical navigation panel on the side of the page
   - Purpose: Secondary navigation or contextual information

5. **Breadcrumb**
   - Description: Path-based navigation showing hierarchy
   - Purpose: Helps users understand their location and navigate back

6. **Pagination**
   - Description: Series of page links for navigating through multi-page content
   - Purpose: Divides content into manageable chunks

7.  **Dropdown Menu**
    - Description: Hidden menu that appears on click/hover
    - Purpose: Provides additional navigation or action options

## Data Display Components

8.  **Card**
    - Description: Bordered box with padding for displaying related content
    - Purpose: Presents information in a clean, contained format

9.  **Table**
    - Description: Rows and columns for structured data presentation
    - Purpose: Displays tabular data in an organized format

10. **Badge**
    - Description: Small count or status indicator next to text
    - Purpose: Shows notifications, statuses, or quantities

11. **Carousel**
    - Description: Rotating display of images or content
    - Purpose: Shows multiple items in limited space

12. **Accordion**
    - Description: Collapsible content sections with headers
    - Purpose: Conserves space while making content accessible

## Data Input Components

13. **Input Field**
    - Description: Text entry fields for collecting user information
    - Purpose: Allows users to enter data in forms

14. **Input Label**
    - Description: Short text next to input field
    - Purpose: Describe the input's purpose

15. **Checkbox/Radio Button**
    - Description: Selection controls for binary or multiple-choice options
    - Purpose: Collects user preferences or selections

16. **Select**
    - Description: Menu of options that expands when clicked
    - Purpose: Space-efficient way to provide multiple choices

17. **Form Group**
    - Description: Container for organizing form elements with labels
    - Purpose: Creates structured, accessible form layouts

## Action Components

18. **Button**
    - Description: Clickable elements styled to indicate interactivity
    - Purpose: Triggers actions or form submissions

19. **Modal/Dialog**
    - Description: Overlay window that appears on top of the main content
    - Purpose: Focuses attention on critical information or actions

20. **Floating Action Button (FAB)**
    - Description: Circular button that floats above the interface
    - Purpose: Provides quick access to primary actions

## Feedback Components

21. **Alert/Notification**
    - Description: Highlighted messages to draw attention
    - Purpose: Communicates important information, warnings, or success messages

22. **Toast**
    - Description: Brief notification that appears temporarily
    - Purpose: Provides non-intrusive feedback about operations

23. **Progress Bar**
    - Description: Horizontal bar showing completion percentage
    - Purpose: Visualizes progress of a process or task

24. **Spinner/Loader**
    - Description: Animated element indicating loading state
    - Purpose: Provides visual feedback during wait times

25. **Tooltip**
    - Description: Small popup that appears when hovering over an element
    - Purpose: Provides additional information without cluttering the interface
